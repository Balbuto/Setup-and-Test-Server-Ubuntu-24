# Contributing

Спасибо за интерес к проекту!

## Как помочь

- Issues: баги, предложения по sysctl, новые тесты
- PR: исправления, документация, обновления SHA256 хэшей для тест-скриптов

## Требования к PR

1. Скрипт должен оставаться **одним файлом** (`server-setup.sh`), без внешних зависимостей кроме стандартных Ubuntu пакетов
2. Целевая система: **Ubuntu 24.04 LTS**
3. Bash строгий режим: `set -Eeuo pipefail`, проверяйте через `shellcheck server-setup.sh`
4. Весь вывод apt - тихий, через `apt_install()` / `apt_update()`
5. Любые изменения в `/etc` - через `backup_file()`
6. Внешние скрипты - только через `secure_download()`, HTTPS
7. Не меняйте дефолты на агрессивные (SAFE профиль должен оставаться консервативным)

## Тестирование

Прогоните базовую настройку на чистом Ubuntu 24.04 VM перед PR.

Проверьте:
- [ ] `shellcheck server-setup.sh` без ошибок
- [ ] Меню 1-6 работают
- [ ] Логи чистые, без ANSI
- [ ] Бэкапы создаются

## Стиль кода

- 4 пробела, без табов
- Функции в snake_case
- Цвета только для вывода на экран, в логи - через `log()` без цветов
- Все apt-вызовы через обёртки

## Сообщение коммитов

Коротко, по-русски или по-английски, как удобно. Префиксы приветствуются: `fix:`, `feat:`, `docs:`.

## Лицензия

Отправляя PR, вы соглашаетесь что ваш код будет под MIT лицензией, как и весь проект.
